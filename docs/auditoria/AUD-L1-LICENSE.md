## Method / Método

Explicitly excluded from the "must have SPDX" set: `examples/` (gitignored upstream clone of RmlUi/gl3w for study/RE only, never compiled into the product, own MIT/public-domain license), all `build*/`/`_deps/`/`.superpowers/` generated/fetched trees (gitignored, not source-controlled), `glintfx/third_party/` third-party vendored headers (carry their own upstream SPDX tag — verified, not ours), `glintfx/patches/*.patch` (a diff against MIT-licensed RmlUi source — correctly carries no SPDX of its own; disclosure lives in `patches/README.md`, which does carry `SPDX-License-Identifier: MPL-2.0` as an HTML comment for the README's own prose). (2) Read `NOTICE` in full and cross-checked every entry against `glintfx/CMakeLists.txt`'s actual `FetchContent`/`find_package`/`target_link_libraries` blocks and `glintfx/third_party/`'s actual file tree, specifically confirming the gl3w removal (v0.8.0) left no trace in either the build graph or `NOTICE`, and that FreeType's `find_package(Freetype REQUIRED)` + `RMLUI_FONT_ENGINE=freetype` + unconditional link (`target_link_libraries(glintfx PRIVATE ... Freetype::Freetype ...)`) matches ADR-0011's "FreeType stays linked either way" decision and `NOTICE`'s FTL entry. (3) `Grep` for `AGPL|Affero` repo-wide, read every hit in context to classify historical-legitimate vs. residual-license-text. (4) Read `LICENSE` end-to-end (374 lines) and compared structurally against the canonical MPL-2.0 v2.0 text (all 10 numbered sections + Exhibits A/B present, Disclaimer of Warranty and Limitation of Liability sections intact and unmodified). (5) Classified each NOTICE-listed dependency's license family against the MPL-2.0 compatibility table. No product file was modified; `TODO.md`/`AUDITORIAS.md` were read only, never edited.

**3. v0.10.0 font-engine change (ADR-0011) reflected coherently / Mudança de motor de fonte da v0.10.0 refletida de forma coerente**

`docs/adr/0011-soft-font-flip.md` records the decision explicitly: "FreeType stays fully linked and supported, either way" — `GLINTFX_OWN_FONT_ENGINE=ON` (new default, own clean-room engine) does **not** drop FreeType; it adds the own engine *alongside* it. Verified in `glintfx/CMakeLists.txt`: `find_package(Freetype REQUIRED)` (unconditional — not gated behind `GLINTFX_OWN_FONT_ENGINE`), `RMLUI_FONT_ENGINE` stays `"freetype"` regardless of the own-engine flag (explicit code comment: "NOT a RmlUi-rebuild switch... do NOT change to 'none'"), and `Freetype::Freetype` remains in the unconditional core `target_link_libraries` block. `NOTICE`'s FTL entry is therefore still accurate and required — confirmed no discrepancy. The ADR itself explicitly scopes the **full/irreversible flip** (dropping FreeType, removing the `NOTICE` FTL entry) as a **separate, future, unscheduled** ADR — this audit finds nothing in the current codebase pre-empting or contradicting that boundary.
