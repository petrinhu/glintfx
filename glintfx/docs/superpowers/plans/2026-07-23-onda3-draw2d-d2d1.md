## Part A -- EN

### 0. What this wave is and the one central risk

D2D-1 is the program's **only G-sized slice**: the public surface gains its first way to
draw a game world (texture load + `draw_sprite` + internal per-texture batching), working in
BOTH consumption modes (`App` frame hook and embed), proven by pixel-readback under Xvfb and
compiled by the MSVC job.

**The central risk, named:** the Draw2D GL path will COHABIT the same GL context as the
RmlUi GL3 renderer (premultiplied alpha, composition on FBO 0, MSAA off under llvmpipe).
The bug class this invites is exactly the `GlStateGuard` class: one renderer leaving state
that silently breaks the other. Section 3 (D9) turns that into an explicit, test-pinned
state contract instead of an accident. Everything else in this wave is comparatively
routine engineering.

**Honest finding this plan adds to the program's scope line:** the program says D2D-1 loads
textures "image atom -> GL, premultiplied", but the image atom has NO standalone boundary
today -- decode+premultiply+256MiB-cap live fused inside `render_gl3.cpp`
(`Gl3RenderInterface::LoadTexture`, ui-side), and `GLINTFX_MODULE_IMAGE` is a reserved flag
forced ON. Draw2D must NOT include ui code, so a small internal carve (slice D2D-1A, section
1) is REQUIRED work, not optional refactoring. Declared here instead of discovered mid-wave.

### 3. Design decisions (D1-D11) -- normative, conditional on section 2 = (1a)+(2a)+(3a)

**D1 -- Module mechanics.** `GLINTFX_MODULE_DRAW2D`, real ON/OFF, default ON, same CMake
block discipline as `GLINTFX_MODULE_AUDIO`/`GAMEPAD` (option + guard + source-list gating).
OFF removes `draw2d.cpp`/`sprite_batch.hpp`/`image_decode.hpp`-consumers' draw2d TU from the
build and every `glintfx::draw2d`-related symbol from `libglintfx.a` (proof: `nm` shows zero
`Draw2d`-mangled symbols with OFF -- the reviewer runs this, ADR-0013 gate style). Depends on
core + the D2D-1A image seam + the internal gl-loader; NOT on window (works in embed hosts
with no GLFW), NOT on ui, NOT on fx. It is compiled in the embed (GLFW=OFF) build and
therefore in the **MSVC job** -- that job becomes the MSVC-compiled gate the program demands.

**D2 -- Own GL pipeline, house substrate.** One GL 3.3 core program (textured quad,
premultiplied blend `GL_ONE, GL_ONE_MINUS_SRC_ALPHA`), one VAO/VBO (dynamic, orphaned per
flush), internal `glx_`-prefixed loader (`gl_loader.h` is private; nothing new is exported --
the `nm` regression check `' [BbDd] gl[A-Z]'` must stay at zero matches). `Draw2d::init()`
calls `glx_gl_load()` itself (required: with ui OFF nobody else loads the table; the loader
is idempotent-safe to call again in App/embed where the engine already loaded it -- implementer
A verifies that claim in code and the reviewer re-verifies; if it is NOT idempotent, guard it
with a local flag rather than assuming).

**D3 -- The image-decode carve (D2D-1A) is minimal and internal.** Extract from
`render_gl3.cpp` into `src/image_decode.hpp`: file-read with the existing 256 MiB
AUD-L1-PARSE cap, `stbi_load_from_memory` forced-RGBA decode, in-place premultiply. The
helper's interface is pure C++ (bytes in, pixels out) with zero GL and zero RmlUi -- callable
and testable headless. `Gl3RenderInterface::LoadTexture` becomes a thin caller (TGA fallback
path untouched). This does NOT make `GLINTFX_MODULE_IMAGE` a real flag (it stays reserved,
forced ON -- flipping it is future atomization work per ADR-0015 (c)'s honesty pattern); it
creates the SEAM Draw2D and the Wave-4 SVG insertion both need. Zero behaviour change
intended; the existing `texture_png_alpha` + `asset_decode_hostile_sanity` suites are the
regression guard.

**D4 -- Frame bracket + batching policy.** `begin(target_w, target_h)` ... N x
`draw_sprite(...)` ... `end()`. The batcher accumulates quads while the texture stays the
same and flushes on texture change, on `end()`, and on reaching capacity (4096 quads --
bounded memory against hostile/degenerate call streams, transparent to the caller).
`draw_sprite` outside a `begin`/`end` bracket: fail-high (log once via the existing dedup'd
log path, no-op). Nested `begin`: log + implicit `end()` of the previous bracket. All
batching DECISIONS live in the pure `src/sprite_batch.hpp` (no GL calls: takes draw commands,
emits vertex data + flush ranges) so the whole policy is unit-testable headless; `draw2d.cpp`
owns only the GL execution of what the pure batcher decided.

**D5 -- Coordinate contract.** Screen-space pixels, top-left origin, y-down -- the SAME space
as `UiEvent` mouse coordinates and `get_element_box` (one coordinate story for the whole
library; the embed-integration section 10 contract extends naturally). Internally an ortho
projection maps to GL clip space (y-flip inside, never the caller's problem). `src_px` is a
sub-rectangle of the texture in TEXEL coordinates (top-left origin, y-down, matching image
memory order), `{0,0,0,0}` sentinel = whole texture. Camera/world space: D2D-2, out.

**D6 -- Composition order (per 3a).** No new ordering API. Documented recipes: App = draw
inside the frame hook (under UI, state auto-restored after the hook -- the v0.19.0 contract,
verbatim); embed = call anywhere the host's GL frame allows, before `ui_layer.render()` for
world-under-UI (the GusWorld production pattern) or after it for overlays (host's choice,
host's state responsibility as with any of its own GL). Post-UI App hook: INBOX seed.

**D7 -- `Texture2d` is a plain value handle, validated internally.** Public struct with
read-only accessors (`ok()`, `width()`, `height()`) and an opaque numeric id documented
do-not-touch. `Draw2d` keeps an internal registry (id -> GL name + generation); `draw_sprite`
with an unknown/stale/tampered id is fail-high (log-dedup'd, no-op) -- never UB, never a raw
GL name trusted from outside. `load_texture(const char* path)`: reads the file itself (plain
ifstream on the given path -- `const char*` overload, avoiding the `filesystem::path::c_str()`
wchar_t MSVC trap), 256 MiB cap, decode via the D3 helper, premultiply, upload
RGBA8 + `GL_LINEAR`/clamp. Failure -> `Texture2d` with `ok() == false` (drawable as no-op).
`destroy_texture(Texture2d&)`: safe on invalid/double (fail-high), zeroes the handle.
`load_texture` before `init()` / after `shutdown()`: fail-high, `ok() == false` result.

**D8 -- Tint semantics (premultiplied, spelled out).** `ColorF` is straight (non-premultiplied)
RGBA in [0,1], clamped on intake. The shader computes
`out = texel_premult * vec4(tint.rgb * tint.a, tint.a)` -- the correct modulation in
premultiplied space (tint alpha fades the sprite as a whole; tint rgb scales the colour).
Default tint = opaque white = identity. This exact formula goes in the doc AND in a
pixel-readback test with a computed expectation (a formula without a delta test that
exercises it is the house's named silent-bug class).

**D9 -- GL-state contract (the anti-`GlStateGuard`-bug decision).** Draw2D sets EVERY piece
of state it depends on at each flush (program, VAO, blend func+equation+enable, active
texture unit + 2D binding; explicitly disables depth test, cull, scissor for its draws) and
assumes NOTHING from the previous frame or the cohabiting renderer. It does NOT restore
state after `end()` -- restoration is the enclosing contract's job, and both enclosing
contracts already exist: in App the frame hook's state is auto-restored before the UI pass
(v0.19.0, `app.hpp` doc); in embed the UI pass re-establishes its own full state inside
`GlStateGuard` regardless of what ran before it, and what Draw2D leaves behind is the host's
to manage exactly like the host's own raw GL today. The full touched-state list is
enumerated in `docs/draw2d.md` (same style as `gl_state.hpp`'s captured list). Pinned by the
coexistence tests (section 5): sprite pass then UI pass, and UI pass then sprite pass, both
render correctly in one frame.

**D10 -- Fail-high input surface (domino rule applied up front, not post-audit).** Validate
floats BEFORE any cast or arithmetic: `dst`/`src` with any NaN/Inf member -> skip + dedup'd
log. `dst.w <= 0 || dst.h <= 0` -> no-op (not an error; degenerate sprite). `src_px` outside
the texture bounds -> clamped (documented; a half-off-atlas frame is a common legit case
under animation math jitter). `begin` with `target_w/h <= 0` -> fail-high bracket (draws
no-op until `end`). Negative coordinates in `dst` are LEGAL (off-screen left/top clipping is
the GPU's job). Every guard listed here gets a hostile test AND its "does the twin exist?"
sweep in review.

**D11 -- Lifetime/RAII.** `Draw2d` is move-only, pImpl, same facade discipline as
`App`/`UiLayer`. `~Draw2d()`/`shutdown()` release GL objects only if `init()` succeeded;
destruction without a current GL context is documented UB-of-the-platform (same honesty as
every GL wrapper; the doc says "destroy while the context that created it is current" --
mirroring what `docs/embed-integration.md` already demands for `UiLayer`).

### 4. Public API contract (literal)

```cpp
// include/glintfx/draw2d.hpp -- NEW public header (module GLINTFX_MODULE_DRAW2D).
// Zero third-party types. EN/PT doc-comments per house style on everything below.
namespace glintfx {

struct RectF  { float x = 0.f; float y = 0.f; float w = 0.f; float h = 0.f; };
struct ColorF { float r = 1.f; float g = 1.f; float b = 1.f; float a = 1.f; }; // straight RGBA [0,1]

struct Texture2d {          // value handle; copyable; opaque id, do not touch
  bool ok() const;
  int  width() const;
  int  height() const;
  // impl: numeric id + generation, validated by Draw2d's registry (D7)
};

class Draw2d {              // move-only, pImpl (house facade discipline)
public:
  Draw2d();
  ~Draw2d();
  Draw2d(Draw2d&&) noexcept;
  Draw2d& operator=(Draw2d&&) noexcept;

  bool init();              // requires a CURRENT GL 3.3 core context; false on failure
  bool ok() const;
  void shutdown();          // idempotent; context must be current (D11)

  Texture2d load_texture(const char* path);   // PNG/JPG/TGA; premultiplied on upload (D7)
  void destroy_texture(Texture2d& tex);       // safe on invalid/double (fail-high)

  void begin(int target_width, int target_height);   // screen px, top-left, y-down (D5)
  void draw_sprite(const Texture2d& tex, const RectF& dst,
                   const RectF& src_px = RectF{},     // {0,0,0,0} = whole texture
                   const ColorF& tint = ColorF{});    // white = identity (D8)
  void end();                                         // flush (D4)
};

} // namespace glintfx
```

`glintfx.hpp` gains the include under the module guard, same pattern as `audio.hpp`.
No method on `App`/`UiLayer` changes; hosting is by construction (D6).

### 5. Testing (what is headless-testable, what is not, downgrades declared)

**Canonical headless line (copy WHOLE, house rule):**
`env -u WAYLAND_DISPLAY XDG_RUNTIME_DIR=/var/tmp/fake_xdg_runtime xvfb-run -a ctest --test-dir <build> --output-on-failure -j1`

1. **Pure units (deterministic, no display, the bulk):**
   - NEW `sprite_batch_sanity`: batching policy pinned -- accumulate same-texture, flush on
     texture change / end / 4096-quad capacity; vertex generation (positions from dst,
     UVs from src_px including the whole-texture sentinel and out-of-bounds clamp, y-down
     texel space); tint clamped to [0,1]; NaN/Inf rejection BEFORE any vertex math;
     degenerate dst no-op; draw-outside-bracket and nested-begin policies; hostile stream
     (100k draws) stays memory-bounded via capacity flush.
   - NEW `image_decode_sanity`: the D2D-1A helper direct -- valid PNG/JPG decode + premultiply
     correctness on known pixels; the existing hostile corpus (truncated, oversized versus
     the 256 MiB cap, malformed) reused at the helper seam.
2. **Pixel-readback under Xvfb/llvmpipe (statistical, `render_sanity` class -- NOT pixel-exact):**
   - NEW `draw2d_render_sanity` (embed fixture, test-only GLFW helper context): solid-colour
     sprite readback (region mean within tolerance); alpha blend of a semi-transparent sprite
     over a known clear colour matches the premultiplied expectation; src_px quadrant of a
     2x2 checker texture shows the right dominant colour; the D8 tint formula verified with a
     computed expected value (delta test for the conversion formula, house rule).
   - NEW `draw2d_ui_coexist_sanity`: one frame with sprite pass THEN `ui_layer.render()`, and
     one with the reverse order -- both readbacks show both layers correct (D9 pinned). This
     is the test that guards the central risk of the wave.
   - NEW `app_draw2d_smoke` (GLFW=ON leg): Draw2d inside `set_frame_callback`, UI document on
     top, readback shows sprite under UI; also pins init-after-App-construction works.
   - NEW `draw2d_hostile_sanity`: load_texture on hostile files (reuse corpus), draw with
     destroyed/tampered handle, load before init, begin(0,0), double shutdown -- all fail-high,
     zero crash, ASan-clean.
3. **Declared downgrades (limits stated instead of faked):**
   - **No pixel-exact goldens** in the gate: llvmpipe tile-threading non-determinism (the
     `golden_test` precedent stands); statistical tolerance asserts only. Real-GPU golden
     remains opt-in.
   - **MSVC = compile+link gate only** (the embed job builds the module); no Windows RUNTIME
     execution exists in CI. Stated in CHANGELOG/doc.
   - **No performance numbers**: the perf budget test is D2D-3 by the leader's closed
     decision. D2D-1 claims correctness, not throughput.
   - No nested-compositor leg needed this wave (no window-manager interaction in scope);
     input/window stress rules are untouched.

## Part B -- PT

### 0. O que é a onda e o risco central

O D2D-1 é a única fatia G do programa: a superfície pública ganha o primeiro jeito de
desenhar o mundo do jogo (textura + `draw_sprite` + batching por textura), nos DOIS modos
(`App` via frame hook e embed), provado por pixel-readback sob Xvfb e compilado pelo job
MSVC. **O risco central, nomeado:** o caminho GL do Draw2D vai COABITAR o mesmo contexto do
renderer GL3 do RmlUi (premultiplied alpha, FBO 0, MSAA off) -- a classe de bug é a do
`GlStateGuard`: um renderer deixando estado que quebra o outro em silêncio. A decisão D9
transforma isso num contrato explícito fixado por teste. **Achado honesto que o plano do
programa não nomeou:** o "átomo image" de que o D2D-1 depende NÃO tem fronteira própria hoje
-- decode+premultiply+teto vivem fundidos no `render_gl3.cpp` e `GLINTFX_MODULE_IMAGE` é flag
reservada forçada ON. O recorte interno mínimo (fatia D2D-1A) é trabalho OBRIGATÓRIO,
declarado aqui em vez de descoberto no meio da onda.
